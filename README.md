
# Артель

> В рамках курсовой работы РТУ МИРЭА, 3 курс  
> Дисциплина: «Разработка клиент-серверных приложений»

## Платформа для локальных мастеров

Артель — это объединение мастеров или творческих людей, которые работают рядом, каждый над своим делом, без общей ответственности, но с общим пространством и духом взаимопомощи.

«Артель» — клиент-серверное веб-приложение для взаимодействия локальных мастеров и покупателей.  
Платформа позволяет размещать handmade-изделия, оформлять заказы и вести коммуникацию между участниками системы.

---

## Основные возможности

- регистрация и авторизация пользователей;
- разделение ролей: покупатель и мастер;
- публикация и управление изделиями;
- просмотр каталога handmade-товаров;
- создание и сопровождение заказов;
- переписка между мастером и покупателем в рамках заказа.

## Установка и запуск

### Локально (npm)

#### 1. Предварительная настройка

1. Установите зависимости во всем проекте (из корневой папки):
   ```bash
   npm install
   ```
2. Создайте файл `server/.env` и добавьте настройки:
   ```env
   PORT=5000
   MONGODB_URI=ваш_адрес_подключения_к_atlas
   JWT_SECRET=ваш_секретный_ключ
   ```

#### 2. Команды запуска (из корневой папки)

- `npm run dev` — запуск сервера и клиента одновременно
- `npm run seed` — заполнение базы данных тестовыми данными
- `npm run dev:server` — запуск только сервера
- `npm run dev:client` — запуск только клиента

После запуска клиент доступен по адресу: `http://localhost:5173`

### Docker (альтернативный способ)


После запуска:
- API: `http://localhost:5000`
- клиент: `http://localhost:8080`

Также можно собирать образы из папок `client/` и `server/` — там есть отдельные `Dockerfile`.

Для запуска тестов (при запущенном сервере):
```bash
cd server
npm run fuzz
```

## Особенности MVP-версии

Проект реализован в формате MVP (Minimum Viable Product).
В текущую версию включены только ключевые функции системы, необходимые для демонстрации клиент-серверного взаимодействия и основной бизнес-логики платформы.
